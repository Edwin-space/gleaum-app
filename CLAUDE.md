# 🗂️ 프로젝트 구조 — 작업 전 반드시 확인

이 레포지토리에는 **두 개의 독립적인 프로젝트**가 공존합니다.
명령을 받았을 때 어느 프로젝트에 해당하는지 먼저 판단하고,
해당 프로젝트의 문서와 규칙만 적용하세요.

---

## 📌 프로젝트 판단 기준

### 1️⃣ 메인 앱 (사용자 앱)
**해당 키워드**: 앱, 프론트, 사용자 화면, 일정, 가계부, 공간, 홈, 로그인(앱), Capacitor, iOS, Android, `src/`

- **소스 루트**: `src/`
- **배포**: `gleaum.com` (Vercel 프로젝트: `gleaum-app`)
- **적용 문서**: 아래 `@AGENTS.md`, `@DESIGN.md`

### 2️⃣ 백오피스 (관리자 어드민)
**해당 키워드**: 백오피스, 어드민, 관리자, 광고 매니저, CRM, 회원 관리, 공간 관리, 릴리즈, 세션 타이머, `backoffice/`

- **소스 루트**: `backoffice/src/`
- **배포**: `gleaum-backoffice.vercel.app` (Vercel 프로젝트: `gleaum-backoffice`)
- **적용 문서**: `backoffice/CLAUDE.md` → `backoffice/docs/` 전용 문서
- ⛔ 루트의 `AGENTS.md`, `DESIGN.md` 규칙 **적용 안 됨**

---

## ⚡ 판단이 모호할 때

작업 대상 파일 경로를 확인하세요:
- 경로가 `backoffice/` 로 시작 → **백오피스 규칙 적용**
- 경로가 `src/` 로 시작 → **메인 앱 규칙 적용**
- 둘 다 언급될 경우 → 각각의 규칙을 분리하여 적용
